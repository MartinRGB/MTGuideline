简约、语境、手势交互性

屏下指纹

  • 关键词 粒子、效果、指纹
  • 设计目的 为用户提供多样的屏下指纹动画，缓解延迟焦虑
  • 设计理念 解锁成功时，提供爆裂散开的感觉
  • 设计反思 效果类动画需要思路清晰，然后动手实现效果。要考虑效果所搭配的背景
  • 开发情况 开发完成

便签

  • 关键词 滚动、转场、模糊
  • 设计目的 结合 6.0 无拘束、留白的理念，尽量提供平滑过渡的转场体验
  • 设计理念 从安卓代码层的入场、转场、滚动等基础动态形式进行思考
  • 开发情况 6.0 仅开发 Demo

OTA 升级

  • 关键词 光效、呼吸
  • 设计目的 结合 6.0 整体视觉感，提供轻量柔和的动画升级体验
  • 设计理念 主要从视觉稿上的 柔和、渐变光感 入手
  • 开发情况 6.0 开发完成

通知中心

  • 关键词 手势驱动、图标动画细节
  • 设计目的 优化旧版本的通知中心体验，同时配合新版本通透的视觉风格提供相应的动态。
  • 设计理念 在通知、快捷操作等高频交互中增强手势操控感，图标动画增加使用愉悦性，缓解焦虑。
  • 开发情况 6.0 开发完成，仍有细节需要优化。

相机

  • 关键词 平滑体验
  • 设计目的 致力于解决老版本相机中，因为 UI 由 OPENGL 绘制导致的体验不一致、平滑的问题
  • 设计理念 为 图形元素 提供更多的动画效果，使用更加舒缓的动画曲线。
  • 开发情况 6.0 开发未完成，后续将持续追踪。

相册

  • 关键词 手势、转场
  • 设计目的 为高频的图片浏览操作增加交互，同时润滑转场的体验。
  • 设计理念 根据相册的空间纵深模型进行设计。
  • 开发情况 6.0 开发未完成，跟相机一样，界面由 OpenGL 硬件绘制层与 Android 软件绘制层共同构成，后续将持续追踪并进行重点攻关。

动态壁纸 - 烟雾

  • 关键词 实验、交互
  • 设计目的 为壁纸添加更多互动可能性，为系统植入 3D 引擎雏形
  • 设计理念 根据贴图映射以及卷曲算法，产生烟雾
  • 开发情况 6.0 开发仍需继续攻关，比如优化细节，降低功耗。
  • 设计反思 该项目阻力较大，一方面配合人员频繁更替，另外一方面技术难度导致意图无法传递。应该意识到，当无法配合时，个人努力上，应该尽可能将项目走到最远，然后进行交付，尤其是这种开发难度高的项目，应该在设计改动完成后再交付开发，保持对设计的控制力。

动态壁纸 - 光流

  • 关键词 渐变、流动感
  • 设计目的 为壁纸添加渐变效果
  • 设计理念 噪音抖动，扩大导致的流动感
  • 开发情况 仍需思考功耗问题应该如何优化。
  • 设计反思 该项目前期思考准备不足，投入度不够，有 Apple Mac OS 的范例以及 Pixel 动态壁纸的范例的珠玉在前，单独渐变、流动感效果似乎过于单薄。

反思、总结与目标

通过一段时间的工作，进行了一系列的反思，总结了以下几点。

  • 在与研发对接的过程中，要尽量与其处于统一语境下，共同思考问题的在框架层的解决方案。（本人目前停留在 应用层 和 demo 中的想当然）
  • 应该量化每个项目的进度，确定完结日期，避免无止境改动。
  • 动态壁纸这类的项目，应该集中、紧凑进行研发，避免项目流程过长，导致质量失控。（针对不同的 RD 应该如何推进项目？）
  • 需要提升 3D、效果类 动画的技术技能。
  • 在全栈设计浪潮下，在设计实践中和主观价值导向下，技术思维和设计思维应该如何取舍？如何互相转化？

下一阶段规划的目标：

  • 协助构建物理动画系统（AE 插件、Android 模块）。
  • 探索 3D 技术（Processing | C4D）在 GUI 中更多的可行性。
  • 重点研究手势驱动的交互的动画改良优化。
